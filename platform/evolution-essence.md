# 演化的本质

CLI 到 Provider 的演化，本质是**"渐进式平台化"**——在保持现有价值的基础上，逐步将能力从边缘（CLI）迁移到中心（Provider），实现治理的规模化。

## 核心本质

### 1. 权力转移：从分散到集中

| 阶段 | 权力分布 | 本质 |
|------|---------|------|
| CLI 时代 | 每个开发者本地决策 | 去中心化治理 |
| 演化过程 | CLI + Provider 双轨运行 | 过渡期妥协 |
| 平台时代 | 服务端统一决策 | 中心化治理 |

**关键洞察**：不是推翻重来，而是让权力"渐进式收拢"。

### 2. 数据流反转：从推送拉取到订阅发布

```
CLI 时代：
开发者 -> 本地修改 -> git push -> 仓库
（开发者主动推送）

平台时代：
开发者 -> 服务端修改 -> 自动同步 -> 仓库
（开发者操作中心，中心分发到边缘）
```

**本质**：从"开发者管理资产"到"开发者操作平台，平台管理资产"。

### 3. 计算迁移：从边缘计算到云计算

| 计算任务 | CLI 时代 | 平台时代 |
|---------|---------|---------|
| 发现 | 本地扫描磁盘 | 服务端扫描多源 |
| 验证 | 本地规则校验 | 服务端策略引擎 |
| 快照 | 本地文件存储 | 服务端历史库 |
| 治理 | 人工检查 | 自动化策略执行 |

**本质**：CLI 从"工具"变成"界面"，Provider 成为"引擎"。

### 4. 治理模式：从自律到他律

| 维度 | CLI 时代 | 平台时代 |
|------|---------|---------|
| 契约规范 | 靠开发者自觉 | 服务端强制校验 |
| 变更审批 | Code Review | 策略引擎自动拦截 |
| 历史追溯 | Git 日志 | 完整治理事件链 |
| 跨项目关联 | 难以实现 | 服务端统一关联 |

**本质**：从"相信人"到"相信系统"。

## 类比理解

| 类比 | CLI 时代 | 平台时代 |
|------|---------|---------|
| **版本控制** | SVN（本地工作副本） | GitHub（中心化仓库 + 社交） |
| **部署** | 手动上传文件 | CI/CD 流水线 |
| **数据库** | SQLite（本地文件） | PostgreSQL（服务端服务） |
| **文档协作** | Word（本地编辑） | 飞书文档（实时协作） |

**共同模式**：从"本地文件"到"在线服务"，从"单机"到"网络效应"。

## 为什么必须渐进？

### 风险规避

| 风险 | 激进方案 | 渐进方案 |
|------|---------|---------|
| 开发者抵触 | 强制切换，放弃 CLI | 双轨运行，逐步引导 |
| 数据丢失 | 一次性迁移，可能丢失 | 增量同步，随时回滚 |
| 系统不稳定 | 大版本替换，Bug 集中 | 小步快跑，快速修复 |
| 网络依赖 | 完全在线，无法离线 | 保留本地模式 |

### 网络效应积累

```
阶段一：少数先锋用户尝试 Provider
    ↓
阶段二：更多用户加入，服务端价值显现（跨项目关联、历史追溯）
    ↓
阶段三：网络效应形成，新用户默认选择 Provider
    ↓
阶段四：CLI 本地模式成为"降级选项"，而非默认
```

## 一句话总结

> **本质是将"个人工具"演化为"组织基础设施"，在保持用户习惯的同时，实现治理能力的规模化、自动化和中心化。**
