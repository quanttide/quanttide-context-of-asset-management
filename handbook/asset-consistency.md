# 资产一致性校验方法

## 什么是资产一致性校验

当同一业务领域存在多份相关文档时，需要确保它们之间的定义、标准、需求保持一致。不一致的文档会导致智能体接收到矛盾的上下文，产生错误的交付物。

## 三类资产及其角色

| 资产类型 | 示例 | 角色 |
|----------|------|------|
| **平台资产** | `apps/qtcloud-asset/.agents/skills/product-brd/SKILL.md` | 产品实际定义的标准、规范、功能 |
| **语境资产** | `docs/context/platform/brd.md` | 为智能体提供工作上下文的参考文档 |
| **产品文档** | `apps/qtcloud-asset/docs/prd/*` | 产品的需求实现，用户故事驱动 |

- 平台资产 = 标准来源（What is）
- 语境资产 = 智能体认知来源（What agent knows）
- 产品文档 = 实现来源（How we build it）

## 校验活动

### 活动一：平台资产 vs 语境资产（同步）

**目的**：确保语境资产中的定义与平台资产中的标准一致。

**方法**：
1. 找到平台资产中定义的标准（如 SKILL.md 中的规范）
2. 对照语境资产中对应的文档
3. 将缺失的标准注入语境资产
4. 提交变更并记录操作

**案例**：将 `product-brd/SKILL.md` 的三改变、问题四要素、假设句式注入 `docs/context/platform/brd.md`

### 活动二：平台资产 vs 产品文档（追溯）

**目的**：验证产品 PRD 是否回应了语境文档中提出的问题。

**方法**：
1. 列出语境文档中的问题/代沟/需求
2. 在产品 PRD 中找到对应的用户故事或功能模块
3. 确认每个问题都有对应的实现
4. 记录映射关系

**案例**：语境 `prd.md` 的三层代沟 → 平台 `docs/prd/*` 的四个模块（graph、harness、pricing、workflow）

## 校验频率

- **平台资产更新后**：检查语境资产是否需要同步
- **产品文档更新后**：检查是否回应了语境文档中的需求
- **定期审查**：建议每次迭代周期执行一次全量对比

## 记录位置

每次校验操作记录到 `meta/` 目录，文件命名格式：`meta/<日期>-<活动简述>.md`
